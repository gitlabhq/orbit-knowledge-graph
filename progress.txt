## Remaining Gaps (ranked by impact)

### P0: DefaultImport resolution (3.1%, 35K imports)
  Root cause: Vue SFC `export default {}` not tracked as ExportedBinding
  Impact: Most .vue component imports fail to resolve
  Fix: Create default ExportedBinding for Vue virtual class

### P1: Cross-file imported function calls from Vue methods
  Root cause: calls to imported functions inside Options API methods
  (e.g., this calls createAlert(), visitUrl()) produce ImportedCall
  but import resolution fails for 58% of paths
  Impact: ~97% of cross-file calls from Vue methods invisible
  Fix: Depends on P0 + continued webpack alias improvement

### P2: mapState/mapActions/mapMutations (Vuex/Pinia)
  Root cause: Runtime-injected methods/getters not recognized
  Impact: Entire Vuex store layer opaque
  Fix: Parse mapState/mapActions calls in Options API

### P3: False CALLS to Variables (52.7% of total CALLS)
  Root cause: ALL references treated as calls, not just invocations
  Impact: 184K false positive edges
  Fix: Filter by reference type (FunctionCall vs PropertyAccess)

### P4: Duplicate edges (22.8%, 255K excess)
  Root cause: Definition ID hash collisions (same FQN+type, different bytes)
  Impact: Edge inflation, slow queries
  Fix: Include byte position in ID hash (tracked by #60)

### P5: Fan-out bug in import resolution
  Root cause: Resolves to file, creates edges to ALL defs, not name-matched
  Impact: Wrong IMPORTS edges
  Fix: Filter resolve_binding results by imported name

### P6: Apollo smart query callbacks not extracted
### P7: Dynamic import() in Vue components not captured
### P8: Mixin composition not modeled as edges
